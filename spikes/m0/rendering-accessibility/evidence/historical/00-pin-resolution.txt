### Previous Discrepancy

The spike CMakeLists.txt previously claimed the ADR pin was "garbage-collected"
and used `8319d69f7286f40b3cacdb2c8216c8a6dd6a239c` instead. This was
incorrect. Both commits exist in the upstream repository; `08b9c553` is
an ancestor of `8319d69f` with exactly 3 commits between them:

  08b9c5539 Fixed error: cast from pointer to integer of different size
          |
          v (3 commits)
  8319d69f7 Clear SDL_PROP_WINDOW_ANDROID_WINDOW_POINTER when ANativeWindow is freed

### Shallow Clone Issue

FetchContent with `GIT_SHALLOW TRUE` fails to fetch `08b9c553` because
this commit is behind the current default branch tip. The CMake
`git clone --depth 1` only retrieves the branch HEAD. Removing
`GIT_SHALLOW TRUE` for SDL3 (full clone) resolves this. The spike
CMakeLists.txt now fetches SDL3 without the shallow restriction.
